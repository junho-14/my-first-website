== 일정 조율 서비스 DB 구조 정리 (T.M.I) ==

사용자 정보 (tmi_users 테이블)
- 로그인 / 회원가입, 마이페이지 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가)
  * 이메일: (로그인할 때 사용, 중복 불가)
  * 비밀번호:
  * 이름:
  * 프로필 이미지:
  * 로그인 방식: (이메일 / 구글 / 카카오 / 애플)
  * 가입일: (언제 가입했는지)

개인 일정 정보 (tmi_schedules 테이블)
- 개인 캘린더, 일정 등록/수정 화면에서 필요한 정보들:
  * 일정 번호: (자동으로 1, 2, 3... 증가)
  * 작성자: (누구의 일정인지 - tmi_users 테이블과 연결)
  * 제목:
  * 시작일시:
  * 종료일시:
  * 종일 여부: (하루 종일 일정인지)
  * 반복 여부:
  * 반복 유형: (매일 / 매주 / 매월, 반복 아니면 빈 값)
  * 반복 종료일: (반복 일정이 아니면 빈 값)
  * 장소:
  * 메모:
  * 색상: (일정 종류 구분용 - 개인일정/중요일정 등)
  * 공개 범위: (공개 / 비공개)
  * 연결 약속: (그룹 약속이 반영되어 생긴 일정인 경우 - tmi_appointments 테이블과 연결, 개인 일정이면 빈 값)
  * 생성일:
  * 수정일: (마지막으로 언제 수정했는지)

그룹 정보 (tmi_groups 테이블)
- 그룹 목록, 그룹 생성/관리 화면에서 필요한 정보들:
  * 그룹 번호: (자동으로 1, 2, 3... 증가)
  * 그룹명:
  * 대표 이미지:
  * 초대 코드: (그룹 참여용 코드, 고유값)
  * 생성자: (그룹을 만든 사용자 - tmi_users 테이블과 연결, 기본 관리자)
  * 생성일:

그룹 멤버 정보 (tmi_group_members 테이블)
- 그룹 목록, 그룹 관리 화면에서 필요한 정보들:
  * 그룹멤버 번호: (자동으로 1, 2, 3... 증가)
  * 그룹: (어떤 그룹인지 - tmi_groups 테이블과 연결)
  * 사용자: (어떤 사용자인지 - tmi_users 테이블과 연결)
  * 역할: (관리자 / 일반멤버)
  * 즐겨찾기 여부: (그룹 목록 상단 고정용)
  * 참여일: (언제 그룹에 들어왔는지)

그룹 약속 정보 (tmi_appointments 테이블)
- 약속 생성, 약속 상세 화면에서 필요한 정보들:
  * 약속 번호: (자동으로 1, 2, 3... 증가)
  * 그룹: (어떤 그룹의 약속인지 - tmi_groups 테이블과 연결)
  * 제목:
  * 시작일시:
  * 종료일시:
  * 장소:
  * 메모:
  * 생성자: (약속을 만든 사용자 - tmi_users 테이블과 연결)
  * 생성일:
  * 수정일: (마지막으로 언제 수정했는지)

약속 참여자 정보 (tmi_appointment_participants 테이블)
- 약속 생성, 약속 상세 화면에서 필요한 정보들:
  * 참여 번호: (자동으로 1, 2, 3... 증가)
  * 약속: (어떤 약속인지 - tmi_appointments 테이블과 연결)
  * 사용자: (참여 대상 사용자 - tmi_users 테이블과 연결)
  * 참여 상태: (참석 / 미정 / 불참)
  * 응답일: (참여 상태를 마지막으로 선택한 날짜)

알림 정보 (tmi_notifications 테이블)
- 알림 화면에서 필요한 정보들:
  * 알림 번호: (자동으로 1, 2, 3... 증가)
  * 수신자: (알림을 받는 사용자 - tmi_users 테이블과 연결)
  * 알림 유형: (그룹초대 / 약속생성 / 약속수정 / 약속취소 / 참여요청)
  * 관련 그룹: (그룹 초대 알림인 경우 - tmi_groups 테이블과 연결, 없으면 빈 값)
  * 관련 약속: (약속 관련 알림인 경우 - tmi_appointments 테이블과 연결, 없으면 빈 값)
  * 내용:
  * 읽음 여부:
  * 생성일:

테이블 연결 관계
- tmi_users → tmi_schedules: 한 사용자가 여러 개인 일정 등록 가능
- tmi_users → tmi_groups: 한 사용자가 여러 그룹 생성 가능
- tmi_users, tmi_groups → tmi_group_members: 한 사용자가 여러 그룹에 소속 가능, 한 그룹에 여러 사용자 소속 가능
- tmi_groups → tmi_appointments: 한 그룹에 여러 약속 생성 가능
- tmi_users → tmi_appointments: 한 사용자가 여러 약속 생성 가능 (생성자)
- tmi_appointments, tmi_users → tmi_appointment_participants: 한 약속에 여러 참여자, 한 사용자가 여러 약속에 참여 가능
- tmi_appointments → tmi_schedules: 하나의 약속이 확정되면 참여자 각각의 개인 일정으로 반영 (약속 1개 → 참여자 수만큼 일정 생성)
- tmi_users → tmi_notifications: 한 사용자가 여러 알림 수신 가능
- tmi_groups, tmi_appointments → tmi_notifications: 그룹 초대 또는 약속 관련 알림과 각각 연결
