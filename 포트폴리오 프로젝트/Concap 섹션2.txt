포트폴리오 Contact 섹션 개발을 프로젝트 생성 규칙에 따라 진행해줘.

Supabase MCP를 사용해서 방명록 데이터베이스를 구축하고,
모든 개발이 완료되면 GitHub Pages로 최종 배포까지 진행해줘.
GitHub Actions 워크플로우를 사용해서 자동 배포되도록 설정하고, 배포 완료 후 접속 가능한 URL을 안내해줘.

⚠️ 중요: Contact 섹션은 Home 탭의 Contact 섹션을 개발하는 것입니다.
새로운 탭을 개발하면 안됩니다.

⚠️ 기존 Contact 섹션과 방명록은 현재 구현되어 있지만,
현재 UI와 구조는 모두 폐기하고 처음부터 다시 개발해줘.

기존 컴포넌트, 레이아웃, 스타일, CSS는 필요한 부분만 참고하고,
전체 디자인과 구조는 아래 기획서를 기준으로 새롭게 구현해줘.

기존 디자인을 유지하거나 일부만 수정하지 말고,
Contact 섹션과 Guest Book을 모두 새로운 UI로 교체해줘.


== Contact 섹션 UI 컨셉 및 느낌 ==

전체적인 느낌
- 깔끔하고 심플한 포트폴리오 스타일
- 여백이 많고 시원한 레이아웃
- 부드럽고 신뢰감 있는 분위기
- 사용자 친화적인 Contact 페이지

이메일/연락처 표시 방법
- 문의 입력 폼과 연락처 정보를 함께 배치
- Email, Phone, Address를 각각 독립적인 카드 형태로 표현
- 아이콘과 제목, 내용을 함께 표시

SNS 링크 스타일
- 원형 아이콘 사용
- Hover 시 아이콘 색상 변경
- 클릭하면 새 창으로 이동

색상 분위기
- 배경: 파스텔톤 하늘색
- 메인 카드: 흰색
- 버튼 및 아이콘: 파란색
- 강조 문구: 주황색
- 깔끔하고 부드러운 분위기

배치 방법
- 좌우 2단 레이아웃

특별히 원하는 스타일
- 참고이미지: 
    - 포트폴리오 프로젝트\Concap1.png
    - 포트폴리오 프로젝트\Concap2.png
    - 포트폴리오 프로젝트\Concap3.png, 
- 큰 흰색 카드 디자인
- 둥근 모서리
- 반응형 레이아웃
- 심플한 Hover 효과
- 화려한 애니메이션은 사용하지 않음

== 방명록에서 받을 정보 논의 ==

기본 필수 정보
- 작성자 이름
- 방명록 내용/메시지

자동 생성 정보
- 방명록 ID
- 작성 날짜 및 시간

연락처 관련 (선택사항)
- 이메일 주소 (비공개로 저장)

추가 정보 (있으면 좋은 것들)
- 나이대 (20대, 30대 등)

재미있는 정보 (개성 있게)
- 한마디 키워드
- 별점 평가

최종 결정된 정보 목록
1. 작성자 이름
2. 방명록 내용/메시지
3. 방명록 ID
4. 작성 날짜 및 시간
5. 이메일 주소 (선택)
6. 나이대 (선택)
7. 한마디 키워드
8. 별점 평가
