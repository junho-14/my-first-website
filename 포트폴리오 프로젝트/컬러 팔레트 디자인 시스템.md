## CSS 변수 정의
```css
:root {
  /* Primary Colors */
  --color-primary: #0F0F0F;
  --color-primary-light: #1A1A1A;
  --color-primary-dark: #050505;

  /* Secondary Colors */
  --color-secondary: #1478FF;
  --color-accent: #00FFA3;

  /* Background Colors */
  --color-bg-primary: #0F0F0F;
  --color-bg-secondary: #1A1A1A;
  --color-bg-tertiary: #202020;

  /* Text Colors */
  --color-text-primary: #FFFFFF;
  --color-text-secondary: #A0A0A0;
  --color-text-muted: #6E6E6E;

  /* Interactive Colors */
  --color-button-primary: #00FFA3;
  --color-button-hover: #00E693;
  --color-link: #1478FF;
  --color-link-hover: #3D91FF;

  /* Status / Highlight Colors */
  --color-live: #FF2D55;
  --color-highlight: #FFD84D;
  --color-border: #2A2A2A;
}
```

## 컬러 사용 가이드

### Primary Colors
- `--color-primary` (#0F0F0F): 페이지 최상단 배경. 다른 요소가 돋보이도록 가장 넓은 면적에 사용.
- `--color-primary-light` (#1A1A1A): 카드, 섹션 구분, 살짝 떠 보여야 하는 표면(elevation)에 사용.
- `--color-primary-dark` (#050505): 모달 오버레이, 그림자와 겹치는 가장 어두운 영역에 사용.

### Secondary Colors
- `--color-secondary` (#1478FF): 이벤트 배너, 프로모션 영역 등 임팩트가 필요한 보조 강조색.
- `--color-accent` (#00FFA3): 브랜드 아이덴티티를 나타내는 시그니처 컬러. 로고, 활성 상태 테두리, 주요 CTA 버튼처럼 반드시 눈에 띄어야 하는 곳에만 제한적으로 사용.

### Background Colors
- `--color-bg-primary`: 전체 레이아웃의 기본 배경.
- `--color-bg-secondary`: 카드, 리스트 아이템 등 콘텐츠 그룹 배경.
- `--color-bg-tertiary`: 사이드바, 헤더처럼 배경보다 한 단계 밝은 구조적 영역.

### Text Colors
- `--color-text-primary` (#FFFFFF): 제목, 본문 등 반드시 읽혀야 하는 핵심 텍스트.
- `--color-text-secondary` (#A0A0A0): 부제목, 설명, 메타 정보.
- `--color-text-muted` (#6E6E6E): 타임스탬프, 조회수 등 우선순위가 낮은 보조 정보.

### Interactive Colors
- `--color-button-primary` (#00FFA3): 주요 액션 버튼(회원가입, 시작하기 등)의 기본 배경.
- `--color-button-hover` (#00E693): 버튼 hover/active 시 약간 어둡게 톤다운.
- `--color-link` (#1478FF): 텍스트 링크 기본 색상.
- `--color-link-hover` (#3D91FF): 링크 hover 시 밝게 처리해 클릭 가능함을 강조.

### Status / Highlight Colors (확장)
- `--color-live` (#FF2D55): 실시간/진행중 상태 뱃지 등 즉각적 주목이 필요한 요소.
- `--color-highlight` (#FFD84D): 이벤트, 포인트, 보상 등 긍정적 강조가 필요한 요소.
- `--color-border` (#2A2A2A): 카드 및 컴포넌트 간 미세한 구분선.

## 반응형 고려사항

### 다크모드 우선 설계
이 팔레트는 다크 배경(`#0F0F0F`)을 기본값으로 설계되었으므로, 라이트모드를 별도로 지원할 경우 아래처럼 `prefers-color-scheme`으로 분기하는 것을 권장합니다.

```css
:root {
  color-scheme: dark;
}

@media (prefers-color-scheme: light) {
  :root {
    --color-bg-primary: #FFFFFF;
    --color-bg-secondary: #F5F5F5;
    --color-bg-tertiary: #EDEDED;
    --color-text-primary: #0F0F0F;
    --color-text-secondary: #4A4A4A;
    --color-text-muted: #8C8C8C;
    --color-border: #E0E0E0;
  }
}
```

### 화면 크기별 고려사항
- **모바일**: 좁은 화면에서는 `--color-accent`, `--color-live` 같은 강조색의 사용 빈도를 줄여 시각적 피로를 낮춘다. 배경과 텍스트의 명도 대비(WCAG AA 기준 4.5:1 이상)는 화면 크기와 무관하게 항상 유지한다.
- **다크 배경 위 강조색**: `#00FFA3`, `#FF2D55`, `#FFD84D`는 모두 채도가 높아 다크 배경에서는 잘 보이지만, 밝은 배경 위에서는 채도를 낮추거나 어두운 변형을 별도로 준비해야 가독성이 유지된다.
- **접근성**: `--color-text-muted`(#6E6E6E)는 배경 `#0F0F0F` 대비 명도비가 낮은 편이므로, 본문이 아닌 보조 정보(타임스탬프 등)에만 한정해서 사용한다.
