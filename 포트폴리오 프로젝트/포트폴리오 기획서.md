## 2. Hero 섹션 컨셉

**참고 사이트**: adhamdannaway.com

adhamdannaway는 "designer / coder" 이중 정체성을 히어로에서 토글로 보여주는 인터랙션이 인상적이었음(레퍼런스 분석 1번 참고). 다만 두 레퍼런스 모두 실제 인물 사진 기반 히어로였고 일러스트를 쓰지 않았으므로, **일러스트 활용은 두 레퍼런스에 없는 독자적 요소**로 가져가는 것.

| 항목 | 적용 방향 |
|---|---|
| 헤드라인 스타일 | 기술중심형 |
| 시각적 요소 | 일러스트 |
| 색상 톤 | 다크 + 초록계열 네온 |
| 차별화 포인트 | 아래 서술 |

**헤드라인 예시안** (기술중심형):
- "Full-Stack Developer, Design-Minded." — 직무 정체성 + 강점을 한 줄로 압축
- 서브헤드: "React와 Figma를 오가며, 화면 너머의 논리와 화면 위의 감각을 함께 만듭니다."

**시각 요소 제안**: 코드 에디터/터미널 느낌의 일러스트 위에 캐릭터(또는 아바타) 배치. 이미 구현된 다크(`#0F0F0F`) + 네온그린(`#00FFA3`) 팔레트가 코딩 테마 일러스트와 자연스럽게 맞물림 — 컬러 팔레트는 이미 `theme.js`/`index.css`에 구현되어 있어 추가 작업 없이 그대로 활용 가능.

**차별화 포인트 (제안)**:
- adhamdannaway의 "designer/coder" 토글 아이디어를 응용해, Hero에 **"Design" ↔ "Develop" 토글**을 배치. 토글 시 배경 일러스트가 Figma 목업 스타일 ↔ 코드 에디터 스타일로 전환되며, 헤드라인도 "감각적으로 설계합니다" ↔ "안정적으로 구현합니다"로 바뀌는 연출. 강점인 "디자인 감각"을 첫인상 단계에서부터 증명하는 장치가 됨.

## 3. About Me 섹션 구성

**참고 사이트**: adhamdannaway.com(진솔한 서사 + 유머러스한 자기 표현) + rafrasenberg.com(가치관 중심 서술)

두 레퍼런스의 접근을 절충: adhamdannaway처럼 **개인 서사를 진솔하게** 풀되, rafrasenberg처럼 **일하는 태도/가치관을 짧고 단단한 문장**으로 못박는 방식.

- **개발 시작 스토리**: 고등학교 때 웹디자인에 관심을 갖게 되었고, 대학은 다른 학과를 졸업했지만 졸업 후 개발자라는 확고한 꿈이 생겨 본격적으로 시작함.
  - 문구 예시: "고등학생 때 우연히 만든 웹페이지 하나가 시작이었습니다. 전공은 다른 길을 갔지만, 졸업 후 그 끌림은 확신으로 바뀌었고 지금은 개발자로서의 길을 걷고 있습니다."
- **핵심 가치관**: 문제 해결 능력, 디자인 감각
  - rafrasenberg 스타일의 단문 선언 예시: "보기 좋은 것과 잘 작동하는 것은 다르지 않다고 생각합니다."
- **개인적 매력**: 게임, 축구, 호기심
  - 부담 없이 스캔되도록 짧은 태그/아이콘 형태로 배치 (adhamdannaway의 유머러스한 스킬 나열 방식 참고)
- **성장 목표**: AI를 활용하여 풀스택 개발자로 성장, 대기업 취업
  - 이 문구는 6번 차별화 전략의 "AI 협업 프로세스 노출" 아이디어와 바로 연결됨 (아래 참고)

## 4. Skill 섹션 차별화

**참고 사이트**: adhamdannaway.com (진짜 역량 %와 개인적 취향 %를 나란히 배치하는 프로그레스바 표현)

**원본 데이터 보정**: 아이디어 문서에 "주력 기술 (5개)"라고 적혀 있지만 실제로는 7개(HTML, CSS, JavaScript, Java, Python, C/C++, Figma)가 나열되어 있었음. 아래는 이를 "코딩 언어/기술 5개(주력) + 확장 2개"로 재구성한 제안 — 라벨과 개수를 맞추고, 강점인 "디자인 감각"을 코딩 스킬과 분리해 더 부각시키는 효과도 있음. Figma 숙련도는 90%로 확정.

| 구분 | 기술 | 숙련도 | 표현 방식 |
|---|---|---|---|
| 주력 5 | HTML | 90% | 프로그레스바 |
| 주력 5 | CSS | 90% | 프로그레스바 |
| 주력 5 | JavaScript | 85% | 프로그레스바 |
| 주력 5 | Java | 75% | 프로그레스바 |
| 주력 5 | Python | 85% | 프로그레스바 |
| 확장 | C/C++ | 90% | 프로그레스바 (보조 그룹) |
| 확장 | Figma | 90% | 디자인 툴 배지로 별도 강조 (프로그레스바 대신 "Design Sense" 카테고리로 분리 추천) |

**제안 이유**: 개발 언어들과 Figma를 같은 척도(%)로 섞으면 오히려 "디자인 감각"이라는 차별화 강점이 다른 스킬바 사이에 묻힘. Figma만 별도 카테고리(예: "Design Tool")로 빼서 시각적으로 튀게 배치하면, 이력서를 훑어보는 채용 담당자의 눈에 "이 사람은 디자인도 한다"는 메시지가 더 확실히 남음.

## 5. 전체 디자인 방향

| 항목 | 방향 |
|---|---|
| 컬러 팔레트 | `컬러 팔레트 디자인 시스템.md` (치지직 기반 다크+네온그린) — **이미 `theme.js`/`index.css`에 구현 완료, 추가 작업 불필요** |
| 폰트 스타일 | 개성있는 |
| 레이아웃 스타일 | 매거진형 |
| 애니메이션 수준 | 화려함 |

**폰트 제안**: 현재 프로젝트는 Roboto 단일 폰트만 사용 중. "개성있는" 느낌을 위해 헤드라인 전용으로 포인트 폰트를 추가하는 방향을 제안 (예: `Space Grotesk`, `JetBrains Mono` 계열 — 코딩 테마와도 잘 어울림). 본문 가독성을 위해 Roboto는 body 텍스트용으로 유지.

**레이아웃 관련 주의사항**: 현재 `docs/design-system.md`의 반응형 규칙은 "데스크톱/태블릿/모바일 모두 1열 그리드"를 원칙으로 하고 있어, "매거진형(비대칭 다단 그리드)" 요청과 상충할 수 있음. 전체를 매거진형으로 바꾸기보다 **Projects/About처럼 콘텐츠가 풍부한 섹션에 한해 2단 비대칭 그리드**를 부분 적용하고, 나머지는 기존 1열 원칙을 유지하는 절충안을 권장. (전면 레이아웃 변경은 별도 논의 후 진행하는 게 안전함)

**애니메이션 제안**: 스크롤 등장(스크롤 위치에 따라 섹션이 fade-up), Hero 토글 전환 애니메이션, 프로젝트 카드 호버 네온 효과(Projects 탭에 이미 구현된 호버 효과와 톤 일치), 스킬 프로그레스바 로드 시 채워지는 애니메이션.

## 6. 차별화 전략

**경쟁자 대비 내 강점**
신입 풀스택 개발자 포트폴리오 대부분은 "기능 구현 목록" 나열에 그치는데, 본인은 (1) 포트폴리오 자체의 비주얼 완성도로 디자인 감각을 실증하고, (2) 프로젝트 설명에 "왜 이렇게 설계했는가"라는 의사결정 과정을 서술해 소통 능력을 보여줄 수 있음. 즉 포트폴리오가 "결과물 전시장"이 아니라 "역량의 증거"로 기능함.

**타겟 어필 포인트**
스타트업/중소기업은 디자이너 리소스가 부족한 경우가 많아, "Figma 시안을 스스로 해석하고 다듬어 구현까지 가져갈 수 있는 개발자"는 실질적인 채용 매력 포인트가 됨. 채용 담당자가 포트폴리오를 30초만 훑어봐도 "이 사람은 협업 비용이 적겠다"는 인상을 받도록, Hero의 Design/Develop 토글과 Skill 섹션의 Figma 분리 배치가 이 메시지를 반복해서 강화함.

**독창적 아이디어**
1. **Before → After 슬라이더**: 각 프로젝트 카드에 "Figma 시안" ↔ "실제 배포 화면"을 좌우 비교 슬라이더로 넣어, 디자인 감각과 구현력을 한 화면에서 동시에 증명. (경쟁자 대부분은 결과 화면만 보여주는데, 시안-구현 과정을 노출하는 포트폴리오는 드묾)
2. **방명록을 "소통 능력"의 증거로 재해석**: 이미 구현되어 있는 Guestbook 섹션을 단순 부가기능이 아니라 "방문자와 실시간으로 소통하는 개발자"라는 서사의 일부로 About Me 섹션에서 명시적으로 언급. rafrasenberg 분석에서 확인한 "댓글 수가 신뢰를 만든다"는 인사이트와 같은 원리.
3. **AI 협업 프로세스 노출**: 성장 목표에 적힌 "AI를 활용하여 풀스택 개발자가 되기"를 그대로 살려, About Me 또는 별도 섹션에 "이 포트폴리오는 Claude Code와 함께 설계·구현했습니다"라는 짧은 메타 소개를 추가. 단순 마감이 아니라 AI 도구를 실무처럼 다룰 줄 아는 개발자라는 인상을 주고, 요즘 채용 시장에서 점점 중요해지는 "AI 활용 역량"을 자연스럽게 어필하는 효과.
