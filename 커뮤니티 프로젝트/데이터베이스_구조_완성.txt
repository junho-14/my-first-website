== 커뮤니티 사이트 DB 구조 정리 (GameHub) ==

사용자 정보 (nv_users 테이블)
- 회원가입 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가)
  * 이메일: (로그인할 때 사용, 중복 불가)
  * 비밀번호:
  * 닉네임:
  * 프로필 이미지:
  * 권한: (일반회원 / 관리자 - 관리자 페이지 접근용)
  * 가입일: (언제 가입했는지)

카테고리 정보 (nv_categories 테이블)
- 게임별 게시판, 게임 평점 등록에 사용:
  * 카테고리 번호: (자동으로 1, 2, 3... 증가)
  * 카테고리명: (RPG, FPS, MOBA, 스포츠, 액션, 어드벤처, 전략, 공포, 인디게임, 모바일게임)

게시물 정보 (nv_posts 테이블)
- 게시물 목록/상세 화면에서 필요한 정보들:
  * 게시물 번호: (자동으로 1, 2, 3... 증가)
  * 제목:
  * 내용:
  * 대표 이미지:
  * 작성자: (어떤 사용자가 썼는지 - nv_users 테이블과 연결)
  * 카테고리: (어떤 게임 게시판인지 - nv_categories 테이블과 연결)
  * 조회수: (조회할 때마다 증가)
  * 작성일: (언제 썼는지)
  * 수정일: (마지막으로 언제 수정했는지)

댓글 정보 (nv_comments 테이블)
- 댓글 화면에서 필요한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가)
  * 댓글 내용:
  * 작성자: (어떤 사용자가 썼는지 - nv_users 테이블과 연결)
  * 게시물: (어떤 게시물에 달린 댓글인지 - nv_posts 테이블과 연결)
  * 부모 댓글: (대댓글인 경우 원본 댓글 번호 - nv_comments 테이블 자기 참조, 없으면 빈 값)
  * 베스트 여부: (베스트 댓글 상단 고정용)
  * 작성일: (언제 썼는지)

게시글 추천 정보 (nv_post_likes 테이블)
- 게시글 좋아요 / 싫어요 기능에 필요한 정보들:
  * 추천 번호: (자동으로 1, 2, 3... 증가)
  * 사용자: (누가 추천했는지 - nv_users 테이블과 연결)
  * 게시물: (어떤 게시물을 추천했는지 - nv_posts 테이블과 연결)
  * 추천 타입: (좋아요 / 싫어요)
  * 등록일:

댓글 좋아요 정보 (nv_comment_likes 테이블)
- 댓글 좋아요 기능에 필요한 정보들:
  * 좋아요 번호: (자동으로 1, 2, 3... 증가)
  * 사용자: (누가 좋아요 눌렀는지 - nv_users 테이블과 연결)
  * 댓글: (어떤 댓글에 눌렀는지 - nv_comments 테이블과 연결)
  * 등록일:

북마크 정보 (nv_bookmarks 테이블)
- 즐겨찾기 / 북마크 기능에 필요한 정보들:
  * 북마크 번호: (자동으로 1, 2, 3... 증가)
  * 사용자: (누가 북마크했는지 - nv_users 테이블과 연결)
  * 게시물: (어떤 게시물을 북마크했는지 - nv_posts 테이블과 연결)
  * 등록일:

태그 정보 (nv_tags 테이블)
- 게시글 태그 기능에 필요한 정보들:
  * 태그 번호: (자동으로 1, 2, 3... 증가)
  * 태그명:

게시글-태그 연결 정보 (nv_post_tags 테이블)
- 게시물 하나에 태그를 여러 개 붙이기 위한 연결 테이블:
  * 게시물: (nv_posts 테이블과 연결)
  * 태그: (nv_tags 테이블과 연결)

게임 평점 정보 (nv_game_ratings 테이블)
- 게임 평점 등록 기능에 필요한 정보들:
  * 평점 번호: (자동으로 1, 2, 3... 증가)
  * 사용자: (누가 평점을 등록했는지 - nv_users 테이블과 연결)
  * 카테고리: (어떤 게임에 대한 평점인지 - nv_categories 테이블과 연결)
  * 평점: (1~5점 등)
  * 등록일:

신고 정보 (nv_reports 테이블)
- 게시글/댓글 신고 기능에 필요한 정보들:
  * 신고 번호: (자동으로 1, 2, 3... 증가)
  * 신고자: (누가 신고했는지 - nv_users 테이블과 연결)
  * 대상 게시물: (신고 대상이 게시물인 경우 - nv_posts 테이블과 연결, 없으면 빈 값)
  * 대상 댓글: (신고 대상이 댓글인 경우 - nv_comments 테이블과 연결, 없으면 빈 값)
  * 신고 사유:
  * 처리 상태: (대기 / 처리완료 - 관리자 페이지에서 확인용)
  * 신고일:

공지사항 정보 (nv_notices 테이블)
- 공지사항 작성 기능에 필요한 정보들:
  * 공지 번호: (자동으로 1, 2, 3... 증가)
  * 제목:
  * 내용:
  * 작성자: (관리자 - nv_users 테이블과 연결)
  * 작성일:

테이블 연결 관계
- nv_users → nv_posts: 한 사용자가 여러 게시물 작성 가능
- nv_users → nv_comments: 한 사용자가 여러 댓글 작성 가능
- nv_categories → nv_posts: 한 카테고리(게임)에 여러 게시물 포함 가능
- nv_posts → nv_comments: 한 게시물에 여러 댓글 작성 가능
- nv_comments → nv_comments: 한 댓글에 여러 대댓글 작성 가능 (자기 참조)
- nv_users, nv_posts → nv_post_likes: 한 사용자가 여러 게시물에 좋아요/싫어요 가능
- nv_users, nv_comments → nv_comment_likes: 한 사용자가 여러 댓글에 좋아요 가능
- nv_users, nv_posts → nv_bookmarks: 한 사용자가 여러 게시물을 북마크 가능
- nv_posts, nv_tags → nv_post_tags: 한 게시물에 여러 태그, 한 태그가 여러 게시물에 사용 가능
- nv_users, nv_categories → nv_game_ratings: 한 사용자가 여러 게임에 평점 등록 가능
- nv_users → nv_reports: 한 사용자가 여러 신고 가능
- nv_users → nv_notices: 관리자가 여러 공지사항 작성 가능
